# Security Policy

## Supported Versions

We only support the latest release of Patience with security updates.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within Patience, please send an email to:

**security@chadbourne.consulting**

Please include the following information in your report:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (if available)

### What to Expect

- **Response Time**: We aim to acknowledge receipt of your vulnerability report within 48 hours.
- **Updates**: We will keep you informed about the progress of addressing the vulnerability.
- **Disclosure**: We ask that you do not publicly disclose the vulnerability until we have had a chance to address it.

### Security Best Practices

When using Patience:

- Keep your installation up to date with the latest release
- Validate and sanitize all configuration files before use
- Use secure credentials management for API keys and authentication tokens
- Review test scenarios for sensitive information before execution
- Follow the principle of least privilege when configuring bot access

Thank you for helping keep Patience and its users secure!
